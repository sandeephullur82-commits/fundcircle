1. AGENT INVITATION FLOW

Owner/Admin Invite Agent

Invite Agent Modal

ONLY SHOW:

1. Agent Email Address
2. Invite Button

DO NOT SHOW:

- Role selector
- Notes
- Phone number
- Name field

ROLE MUST AUTO ASSIGN:
role = "agent"

---

AGENT EMAIL VALIDATION RULES

Before sending invitation:

CHECK:

1. Email already exists in agents table?
   → Block invitation

Error:
“This email is already registered as an agent.”

---

2. Email already exists in customers table?
   → Block invitation

Error:
“This email is already used by a customer account.”

---

3. Email already invited and pending?
   → Block duplicate invitation

Error:
“Invitation already pending for this email.”

---

4. Email belongs to organization owner/admin?
   → Block invitation

Error:
“This email belongs to an organization administrator.”

---

AFTER VALIDATION

System creates:

- invitationToken
- organizationId
- role = agent
- invitationStatus = pending
- invitedBy
- expiresAt

Send secure invitation email.

---

When agent opens invite link:

Auto-fill:

- email

- email field

- full name
- password

- profile photo
- assigned region

After signup:

- create agent profile
- activate organization access
- assign permissions
- create audit logs

AGENT DATABASE RULES

CUSTOMER VALIDATION RULES

   → Block

   → Block

“This email belongs to an administrator account.”

---

3. Customer already exists?
   → Prevent duplicate customer

“Customer already exists.”

---

4. Phone number already exists?

Error:
“Phone number already registered.”

---

5. Assigned agent exists and active?
   → Required

Error:
“Assigned agent is not active.”
