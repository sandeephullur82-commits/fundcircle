Fix the customer limit calculation and phone number validation in the FundCircle platform.

==================================================
ISSUE 1 — INVALID CUSTOMER LIMIT COUNT

PROBLEM:
The dashboard shows:

"Customer limit reached (0/-1)"

even though no customers were created.

==================================================
ROOT CAUSE

The subscription limits or usage values are:

- undefined
- null
- missing
- negative

The system is calculating:

0 / -1

instead of valid plan limits.

==================================================
FIX REQUIRED

When organization document is created,
initialize default plan limits correctly.

==================================================
DEFAULT PLAN STRUCTURE

organizations/{orgId}

{
plan: "free",

limits: {
maxAgents: 1,
maxCustomers: 25
},

usage: {
activeAgents: 1,
activeCustomers: 0
}
}

==================================================
IMPORTANT VALIDATION

NEVER allow:

- undefined
- null
- negative numbers

==================================================
SAFE FALLBACKS

Use safe defaults everywhere:

const maxCustomers =
organization?.limits?.maxCustomers || 25

const activeCustomers =
organization?.usage?.activeCustomers || 0

==================================================
LIMIT CHECK

const isLimitReached =
activeCustomers >= maxCustomers

==================================================
DISPLAY FIX

Dashboard must show:

Customers:
0 / 25

NOT:
0 / -1
